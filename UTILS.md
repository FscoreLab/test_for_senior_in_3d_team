# Вспомогательные инструменты

Для работы с КТ предоставлен модуль **`utils.py`** на основе **SimpleITK** (работа с `sitk.Image`, геометрия в физическом пространстве).

Использовать его не обязательно — это лишь готовая обвязка, чтобы не тратить время на рутину ввода-вывода и геометрии. Свой пайплайн можно собрать и иначе. Например, использовать библиотеку Monai

## Что внутри

### Чтение и сохранение

- `read_dicom_series` / `read_nifti` — чтение КТ из DICOM-серии или NIfTI (значения в HU);
- `sitk_image_to_numpy` — перевод в numpy-массив;
- `numpy_mask_to_sitk_with_ct_info` — сборка маски с геометрией от КТ;
- `save_mask_with_ct_info` — сохранение маски в NIfTI с правильной геометрией.

### Предобработка

- `apply_brain_ct_window` — мозговое CT-окно (WL=40, WW=80) с нормализацией;
- `resample_to_spacing`, `resample_ct_to_isotropic`, `resample_mask_to_isotropic` — приведение к заданному / изотропному spacing (для КТ — линейная интерполяция, для масок — ближайший сосед).

### Геометрические преобразования (для выравнивания головы)

- `image_center_physical` — физический центр объёма (центр вращения);
- `resample_image_with_transform` — применение `sitk.Transform` с автоматическим расширением кадра, чтобы не обрезать объём;
- `rotation_matrix_to_transform` — обёртка матрицы 3×3 / 4×4 в трансформ.

## Важно про координаты и spacing

- Геометрия (spacing, направление, origin) задаётся в **физическом пространстве** (мм). Если считаете углы, плоскости, расстояния или аффинные преобразования — не работайте только в воксельных индексах без учёта spacing.
- Перед 3D-расчётами удобно привести КТ и маски к **изотропному spacing** (`resample_*_to_isotropic`): тогда расстояния по осям становятся сопоставимыми.
- Порядок осей различается: после `sitk.GetArrayFromImage` массив идёт в `[z, y, x]`, тогда как `spacing` / `size` / `origin` в SimpleITK — в `(x, y, z)`.
- При повороте проверяйте **направление** на одном примере: правильный вариант выпрямляет голову, а не доворачивает сильнее (подробности — в docstring соответствующих функций).

В конце файла есть блок `Example usage` с типовым сценарием: чтение → изотропный spacing → окно → работа с маской → выравнивание головы.
